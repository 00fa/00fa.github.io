修饰符

发表于 2019-11-25 | 分类于 Java ， 基础

分为权限修饰符及特征修饰符两大块，描述其使用的特性

权限修饰符

权限修饰符可以用来修饰 类本身 和类中的成员（除了程序块）

但用来修饰类的时候只有两个可以用 （public 和 默认不写，使用public时候类的名字必须与文件的名字一样，否则编译报错，目的是让生成的class文件跟java文件名字一样）

public 本类 同包 公共的 子类（当前工程内任意类中只要有对象都可以访问）
protected 本类 同包 保护的 子类（通过子类对象在子类范围里访问）
默认不写 本类 同包 默认的 ×
private 本类 × 私有的 ×

特征修饰符

特征修饰符 意义
final 最终的 不可更改的
static 静态的
abstract 抽象的
native 本地的
*transient 瞬时的 短暂的—–序列化
*synchronized 同步的 ——线程问题
*volatile 不稳定的

final 最终的 不可更改的

  • 修饰变量

如果在定义变量时没有赋初始值，给变量一次存值的机会（因为变量在栈内存 内，没有默认值，如果不给机会，那就没意义了），一旦变量存储了值，则不让更改，相当于常量。

如果final修饰的变量是基本数据类型，则变量内的值不让更改 常量
如果final修饰的变量是引用数据类型，则变量内的地址引用不让更改 对象唯一

  • 修饰属性

因为属性如果没有赋初始值，是有默认值存在的，所以属性用final修饰后必须要立即给属性赋初始值，否则编译报错（不像变量还有一次机会），其他跟修饰变量一样（注意引用数据类型）

  • 修饰方法

方法是最终的方法，不可以被子类方法重写（覆盖）

  • 修饰类本身

类是最终的类了，此类不可以被其他子类继承了（太监 无子），一般是一些定义好的工具类（不想被其他人更改）

static 静态元素

  1. 修饰范围 ：

    • 修饰属性
    • 修饰方法
    • 修饰块
    • 修饰类（具体是内部类）

  2. 特点：

    • 静态元素在类加载时就初始化了

    • 静态元素存储在静态元素区，每个类有自己的区域，与其他类不冲突

    • 静态元素值只加载一次

    • 可以直接通过类名字访问，不需要创建对象

    • 静态元素区Garbage Collection无法管理，可以粗暴的认为常驻内存

    • 非静态成员（堆内存里）可以访问静态成员（静态区），反之不行
