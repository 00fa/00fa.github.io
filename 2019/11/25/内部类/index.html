内部类

发表于 2019-11-25 | 分类于 Java ， 基础

内部类的定义，及四种内部类的使用差异。

内部类

指的是java中可以将一个类定义在另一个类的内部

  1. 定义

  2. 内部类可以定义在类的内部（与类成员层次一致）

  3. 内部类可以定义在方法/块内部（与类成员相差一个层次，相当于跟方法的局部变量一个层次）

  1. 成员内部类

    将一个类直接定义在类的里面，作为成员，与属性 或方法层次一致
    成员内部类可以使用不同权限修饰符（不像正常类只有public跟无）
    内部类存在时，源代码编译会产生一个字节码 Demo$InnerDemo.class（中间有$）

    好处：

    1. 可以省略一个.Java文件
    2. 成员内部类可以访问外部类的所有成员 包括私有的
    3. 若想要利用内部类调用外部类的成员 需要 外部类.this.外部类成员

  1. 局部内部类（不常用）

    将一个类定义在方法/块里面，作为成员的内部结构，与临时的局部变量一个层次
    会像一个局部的变量一样，不能用public protected private以及static
    只能用abstract或final
    局部内部类命名规则 Demo$1InnerMethod Demo$2InnerMethod (用数字区分两个重名的)
    局部内部类使用的变量只能是final修饰的，java8默认自带final 深入理解局部内部类访问局部变量为什么必须加final关键字

  1. 匿名内部类

    成员匿名内部类
    局部匿名内部类

    通常接口或抽象类的具体子类这样写，因为开发中可以省略一个类文件
    匿名内部类只有类的体，没有类的所有结构（修饰符 名字 继承 实现）
    所以不能用任何修饰符来修饰 匿名内部类也没有构造方法
    应用：Swing 做一个按钮 绑定一个事件监听器

  2. 静态内部类

    成员静态内部类
    不需要外部类对象创建了，通过正常的方式直接创建内部类（但同样要导包）
    普通的成员内部类 InnerDemo a = outer.new InnerDemo();
    静态的成员内部类 InnerDemoStatic a = new.InnerDemoStatic(); (少了外部类对象.)

    静态元素不能访问非静态成员（自己类跟外部类）
